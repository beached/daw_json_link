211 /***
212 * Custom JSON types can be Strings(default), unquoted Literals, or a mix
213 * String - Parser always expects a JSON string. Will surround serialized
214 * value with double quotes
215 * Literal - Parser will expect a valid JSON literal number, bool, null
216 * Any - Experimental - Parser will return any valid JSON value excluding
217 * leading whitespace. strings will be quoted. Any is suitable for
218 * constructing a json_value to allow adhock parsing if json_raw is not
219 * suitable
